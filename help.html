Anleitung

Ziel dieser Anwendung ist es, Methodenaufrufe inklusive Signatur, Art und Klasse zu verstehen. Hierzu stehen zwei Modi zur Verfügung, die nachfolgend erklärt werden.
In beiden Modi können einzelnen Zeilen angeklickt werden. Fall in dieser Zeile (mindestens) ein Methodenaufruf enthalten ist, so werden weitere Informationen angezeigt.

Modus Browse

Falls in dieser Zeile ein Methodenaufruf stattfindet, so werden alle Informationen zu dem/den Aufruf(en) angezeigt.

Modus Quiz

Falls in dieser Zeile ein Methodenaufruf stattfindet, so werden alle Informationen zu dem/den Aufruf(en) abgefragt. Sobald alle Informationen eingegeben wurden, können die Eingaben mit dem Check Button auf Richtigkeit überprüft werden.

Hinweise

  • Falls in einer Signatur Klassen enthalten sind aus dem Package "java.lang" wie z.B. "java.lang.String", so wird als Eingabe die verkürzte Version "String" akzteptiert.
  • Methodenaufrufe sind immer nur einer Zeile zugewiesen, auch wenn sich diese über mehrere Zeile erstrecken. Es ist immer die erste Zeile, in der der Code für einen solchen Aufruf beginnt. In folgendem Beispiel ist es auch die erste Zeile, obwohl der eigentliche Aufruf in der zweiten Zeile steht.
    public static final Rational MAX_VALUE = new Rational(Integer.MAX_VALUE, 1);
  • Es werden auch implizite Methodenaufrufe angezeigt. Wie z.B. super()-Aufrufe oder Autoboxing. Diese sind für Anfänger schwer zu erkennen, da sie implizites Wissen voraussetzen.
  • Falls mehrere Methodenaufrufe innerhalb einer Zeile stehen, so werden diese chronologisch sowohl im Browse Modus angezeigt, als auch im Quiz Modus abgefragt.
